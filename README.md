## 객체 지향 설계 (SOLID)
* SRP: 한 클래스는 하나의 책임만 가져야 한다.
* OCP: 소프트웨어 요소는 확장에는 열려 있으나 변경에는 닫혀 있어야 한다.
* LSP: 프로그램의 객체는 프로그램의 정확성을 깨뜨리지 않으면서 하위 타입의 인스턴스로 바꿀 수 있어야 한다
* ISP: 특정 클라이언트를 위한 인터페이스 여러 개가 범용 인터페이스 하나보다 낫다
* DIP: 추상화에 의존해야지, 구체화에 의존하면 안된다
